## Code hygiene

### 1. Three remaining `tee -a "$LOG"` calls in setup.ps1 Step 8b

**Where:** setup.ps1 lines 1189, 1192, 1195 (daemon-reload, enable, restart commands inside `$gatewayInstall`).

**What:** Each pipes through `tee -a "$LOG"` where `$LOG = /tmp/openclaw-install.log`. The file is owned by root from Step 8 install.sh and these lines run as clawuser, so the tees fail with "Permission denied." Failures are masked by `|| true`.

**Why it matters:** wasted I/O (writes to a file they can't write); the failure isn't visible because of `|| true`. Cosmetically inconsistent with the install command above which dropped its tee in commit [TBD].

**Fix:** drop the tees entirely OR redirect to a clawuser-owned path like `~/.openclaw/logs/install.log`. Output is already captured by the Windows-side install.log, so dropping is fine.

**Priority:** low. Cosmetic.

### 2. clawfactory-tunables.conf drop-in is redundant on 2026.4.27

**Where:** setup.ps1 Step 8b sub-block (c) writes `~/.config/systemd/user/openclaw-gateway.service.d/clawfactory-tunables.conf` with `TimeoutStartSec=infinity`.

**What:** `openclaw gateway install --force` (called later in $gatewayInstall) auto-writes its own `insecure-loopback.conf` with the same `TimeoutStartSec=infinity`. Both drop-ins coexist because systemd merges all `*.conf` files in the drop-in dir.

**Why it matters:** redundant configuration. Not harmful (systemd handles duplicates correctly) but documents a workaround that's no longer needed on this version.

**Fix options:**
- **A:** delete sub-block (c) entirely. Trust openclaw's own drop-in.
- **B:** keep sub-block (c) as forward-compatibility insurance (in case a future openclaw version stops writing the drop-in itself).

**Recommended:** Option B with a comment explaining the redundancy. Cost is zero, defense-in-depth value is non-zero.

**Priority:** low. Document and move on.

## Distribution / discoverability

### 7. Code signing certificate (Sectigo OV ~$200/yr)

**Where:** ClawFactory-Secure-Setup.iss has the SignTool line commented out. Currently we ship unsigned, so SmartScreen warns users on every install.

**Why it matters:** unsigned executables get aggressive Windows Defender SmartScreen warnings ("Windows protected your PC") that scare non-technical users away. Signing fixes this.

**Fix:** purchase Sectigo OV certificate, configure SignTool in Inno Setup IDE, enable the SignTool line in .iss.

**Priority:** medium. Required prerequisite for Velopack auto-update (item #9). Defer until first revenue.

## Reliability / resilience

### 9. Auto-update via Velopack

**What:** add Velopack-based update mechanism so customers get bug fixes without redownloading the .exe.

**Why it matters:** every bug fix currently requires customer to find new .exe and reinstall. Cumbersome.

**Fix:** integrate Velopack. Two-stage model: pin shipped version, auto-fetch updates from your server after validation.

**Prerequisites:** code signing cert (#7), our own update server, Velopack integration.

**Priority:** v1.x feature. Significant scope.

### 10. Defender exclusion for npm cache

**What:** add `Add-MpPreference -ExclusionPath` for the WSL npm cache path during Step 6 / Step 8.

**Why it matters:** Defender real-time scan adds 50-100% to npm install time. On a cold install, that's 5-10 extra minutes.

**Fix:** few lines of PowerShell. Need to handle the case where Defender isn't the active AV.

**Priority:** low-medium. Quality-of-life improvement.

---

### 11. Retry loop on `wsl --install`

**What:** Step 2 currently calls `wsl --install` once. If network is flaky (we hit 0x80072eff once during testing), the install fails and the user has to restart from scratch.

**Fix:** wrap in a 3-attempt retry with exponential backoff.

**Priority:** medium. Real customer environments have flaky networks.

---

### 12. Schema-rename detection for `openclaw config set` calls

**What:** when openclaw renames a config path between versions, our `openclaw config set` calls silently fail with "path not found." We discover this only when something downstream breaks.

**Fix:** make every `openclaw config set` call check the exit code and log a CLEAR error if the path doesn't exist. Could also pre-validate paths against `openclaw config schema` output before attempting the set.

**Why it matters:** Future-Bret bumps OPENCLAW_VERSION, an unnoticed schema rename happens, install "succeeds" but a feature is broken.

**Priority:** medium. Defense-in-depth against the FIX 2 class of bug recurring.

---

### 13. Surface post-install warnings in wizard UI

**What:** if post-install.ps1 exits non-zero, the wizard currently completes silently as "Setup completed successfully" — the user has no visibility into the WARN.

**Fix:** add a yellow-banner page after the install if any [WARN] lines appear in install.log. Link to the log file path.

**Priority:** medium. User-facing transparency.

### 14. Track upstream OpenClaw fixes

**What:** when openclaw closes the GitHub issues we patched against (#72355, #64928, #18502, #44571, #12003, #65184), our wrappers become defense-in-depth. Should be marked as such in code comments and tracked here.

**Fix:** periodic check (quarterly?) of referenced issue IDs. If closed, update comment to "FIX retained as defense-in-depth — upstream resolved in <commit>".
